# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in StatJam, please report it responsibly:

1. **Do not** open a public GitHub issue for security vulnerabilities.
2. **Email** the maintainers with a clear description of the issue, steps to reproduce, and any impact assessment.
3. We will acknowledge your report and work with you on a fix and disclosure timeline.

We appreciate the security research community’s efforts to help keep StatJam safe.
